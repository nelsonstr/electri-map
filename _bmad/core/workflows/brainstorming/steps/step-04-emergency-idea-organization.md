## EMERGENCY CATEGORIZATION SCHEMA:

### By Priority Level

| Priority | Definition | Timeline | Life-Safety Impact |
|----------|-----------|----------|-------------------|
| **CRITICAL** | Immediate action required | 0-6 hours | Direct threat to life |
| **HIGH** | Urgent implementation | 6-24 hours | Significant operational impact |
| **MEDIUM** | Short-term planning | 1-7 days | Process improvement |
| **LOW** | Strategic enhancement | 1-4 weeks | Capability building |

### By Response Phase

| Phase | Description | Focus Areas |
|-------|-------------|-------------|
| **Preparedness** | Pre-incident planning and training | Prevention, readiness, resource positioning |
| **Response** | Active incident management | Life safety, incident control, initial assessment |
| **Recovery** | Post-incident restoration | Service restoration, community support, debrief |
| **Mitigation** | Long-term risk reduction | Infrastructure hardening, policy changes, system improvement |

### By Agency Type

| Agency Category | Primary Responsibility | Coordination Needs |
|----------------|----------------------|-------------------|
| **Fire Service** | Fire suppression, hazmat, rescue | Fire command integration, mutual aid |

| **EMS** | Emergency medical services | Medical incident command, hospital liaison |
| **Police** | Security, traffic, evacuation | Law enforcement coordination, scene security |
| **Multi-Agency** | Joint operations | Unified command, information sharing |

### By Resource Need

| Resource Level | Definition | Implementation Approach |
|----------------|-----------|----------------------|
| **Low** | Existing resources only | Immediate deployment possible |
| **Medium** | Resource reallocation | Requires authorization, 6-24 hour lead time |
| **High** | New investment required | Budget approval, procurement process |
| **Critical** | Emergency procurement | Exigent circumstances, accelerated acquisition |

## EMERGENCY PRIORITIZATION METHODS:

### 1. Impact-Urgency Matrix

```
                    HIGH URGENCY
                        │
    ┌───────────────────┼───────────────────┐
    │                   │                   │
    │   DO FIRST        │   SCHEDULE        │

    │                   │                   │
    │ • Life-safety     │ • Operational     │
    │   immediate       │   improvements     │
    │ • Incident        │ • Process fixes    │
    │   response        │ • Quick wins       │
    │                   │                   │
LOW ├───────────────────┼───────────────────┤ HIGH
IMPACT                   │                   IMPACT
    │                   │                   │
    │   DELEGATE        │   LATER           │
    │   (Medium)        │   (Low Priority)  │
    │                   │                   │
    │ • Training        │ • Nice-to-have    │
    │ • Documentation   │ • Enhancements    │
    │ • Planning        │ • Future projects │
    │                   │                   │
    └───────────────────┼───────────────────┘
                        │
                    LOW URGENCY
```

**Application Protocol:**
- **Quadrant 1 (Do First):** Immediate action, assign dedicated resources
- **Quadrant 2 (Schedule):** Plan within 24-48 hours, allocate resources
- **Quadrant 3 (Delegate):** Assign to team, schedule within week
- **Quadrant 4 (Later):** Document for future consideration

### 2. MoSCoW for Emergency Services

| Category | Definition | Emergency Application |
|----------|-----------|---------------------|
| **Must Have** | Non-negotiable for life-safety | Immediate implementation, no alternatives |
| **Should Have** | Important but not critical | Implement if resources permit |
| **Could Have** | Desirable enhancement | Implement if easy/low cost |
| **Won't Have** | Deferred to future | Document for later consideration |

**Emergency MoSCoW Criteria:**
- **Must Have:** Direct life-safety impact, regulatory requirement, critical path dependency
- **Should Have:** Operational improvement, efficiency gain, quality enhancement
- **Could Have:** Nice-to-have capability, cosmetic improvement, convenience feature
- **Won't Have:** Out of scope, resource prohibitive, low priority

### 3. Speed-Impact Analysis

| Speed Category | Definition | Implementation Approach |
|---------------|-----------|----------------------|
| **Quick Wins** | <4 hours to implement | Immediate execution |
| **Fast Track** | 4-24 hours | Accelerated deployment |
| **Standard** | 1-7 days | Normal implementation |
| **Strategic** | >1 week | Phased implementation |

**Evaluation Criteria:**
- **Speed Factors:** Resource availability, approval requirements, complexity
- **Impact Factors:** Life-safety improvement, operational benefit, risk reduction
- **Trade-off Analysis:** When speed and impact conflict, life-safety prevails

### 4. Risk-Based Ranking

**Risk Assessment Matrix:**

| Risk Level | Probability | Impact | Priority |

| **High** | Medium | Major | #3 |
| **Medium** | High | Moderate | #4 |

| **Low** | Any | Minor | Document only |

**Risk Factors for Emergency Services:**
- Life-safety exposure
- Operational continuity
- Reputation/communication impact
- Legal/regulatory compliance
- Resource consumption
